## IOS-作业

由于本地没有MAC条件，所以这里就不贴实际的代码了。

其实这个实现有很多中方法，最简单的能够想到的就是将这些数据保存在文件中，那么如何保存呢，又如何读取呢？

一个当然是自己写轮子，写一个满足需求的文件读取和文件写入函数，第二个就是使用轮子了，根据老师的提示，我们可以使用`NSUserDefault`进行持久化数据，不过相比较要让自己来写，使用NSUserDefault简单的多，并且还可能高效的多，下面讲讲大致的思路：

首先，当登录界面进行渲染（`viewDidLoad`）的时候，我们可以从持久化的数据中使用`objectForKey`进行读取由于`NSUserDefault`的实现机制，即使键值不存在也不会报错，只会返回空值，所以我们不用进行判断(对这个底层的不是很清楚，如果渲染的花销，大于这个判断的花销，那么最好还是进行判断一下，如果对应的key不存在，那么就不要修改`UITextField`元素中的值)，当转到另外一个消息列表控制层(controller)的时候(同样应该为`viewDidLoad`)，我们可以使用`setObject`将房间号，用户名进行持久化。
